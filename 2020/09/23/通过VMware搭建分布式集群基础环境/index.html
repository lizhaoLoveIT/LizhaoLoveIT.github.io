通过VMware搭建分布式集群基础环境

字数总计: 582|阅读时长: 1 分钟

软件安装包：VMware15.5.1 CentOS-7-x86_64-DVD-1908 jdk-8u251-linux-x64.tar.gz

安装好软件后

新建虚拟机

上述路径可在编辑首选项中设置

选择网络时选择使用桥接网络，后面说好处。

激活网卡并设置静态ip

虚拟机静态ip桥接模式好处

  1. 可以上外网
  2. 和宿主机处于同一网段，方便和同一局域网内网电脑连接(可以将宿主机当纯服务机器用，开发在另一台机器)
  3. 固定ip，方便查看服务

编辑->虚拟网络编辑器

桥接到的网卡必须和宿主机网络网卡信息一致。而且，如果要配置静态ip，宿主机也必须是静态 ip。否则虚拟机的网络将不可用。

查看宿主机 ip 设置 ipconfig /all

对比修改网络配置文件 /etc/sysconfig/network-scripts 来激活网卡，要和宿主机保持一致

改完后重启网络就可以访问外网了

再 ping baidu.com 就 ping 通了。接下来下载工具包，查看ip方便 yum install net-tools

修改主机 hostname

因为服务器是用来做集群的，所以需要给每个主机命名

安装 JDK

  • 查看系统是否自带 JDK java -version
  • 查看相关 java 文件 rpm -qa | grep java
  • 删除相关文件
  • 下载 JDK

克隆虚拟机

我们已经配置好了一台静态ip、安装有JDK基础环境的虚拟机。如果集群有多台虚拟机，肯定不会再从头来一遍浪费时间。
可以通过 VMware的克隆功能进行快速扩充新的虚拟机。

克隆前提示关闭虚拟机

创建快照过程是非必须的，但为了后面创建虚拟机最好还是有这个步骤。

以上就克隆了一台，速度还是很快的。再配置一台

调整配置

现在有3台一模一样配置的主机，作为一个基本的可用集群，需要进行一些细微调整。

需要将 s1 s2 的ip改为 192.168.0.101 和 192.168.0.102，然后再修改 hostname，重启机器。

最终，一个小的集群就搭建完毕了。
